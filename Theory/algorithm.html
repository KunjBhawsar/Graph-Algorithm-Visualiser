Algorithm Details

1. Breadth-First Search (BFS)

BFS is a graph traversal algorithm that explores vertices in order of their distance from the source vertex. It uses a queue data structure to keep track of vertices to visit next.

How it Works:

  1. Start at the source vertex and mark it as visited
  2. Enqueue the source vertex
  3. While the queue is not empty:
    • Dequeue a vertex
    • For each unvisited neighbor:
      • Mark it as visited
      • Enqueue it
Time Complexity: O(V + E) where V is vertices and E is edges
Space Complexity: O(V)

Applications:

  • Finding shortest path in unweighted graphs
  • Level order traversal of trees
  • Web crawlers
  • Social networking connections

2. Depth-First Search (DFS)

DFS explores as far as possible along each branch before backtracking. It uses a stack (or recursion) to keep track of vertices.

How it Works:

  1. Start at the source vertex and mark it as visited
  2. For each unvisited neighbor:
    • Recursively visit that neighbor
  3. Backtrack when no unvisited neighbors exist
Time Complexity: O(V + E)
Space Complexity: O(V) for recursion stack

Applications:

  • Detecting cycles in graphs
  • Topological sorting
  • Finding strongly connected components
  • Maze solving

3. Prim's Algorithm

Prim's algorithm is a greedy algorithm that finds a minimum spanning tree for a weighted undirected graph. It starts with a single vertex and grows the MST one edge at a time.

How it Works:

  1. Start with any vertex and add it to MST
  2. Add all edges from this vertex to the priority queue
  3. While MST doesn't contain all vertices:
    • Extract minimum weight edge from priority queue
    • If the edge connects to a new vertex:
      • Add the edge to MST
      • Add the new vertex to MST
      • Add all edges from new vertex to priority queue
Time Complexity: O(E log V) with binary heap
Space Complexity: O(V)

Applications:

  • Network design (laying cables, pipes)
  • Approximation algorithms for NP-hard problems
  • Cluster analysis

4. Kruskal's Algorithm

Kruskal's algorithm is a greedy algorithm that finds a minimum spanning tree by sorting all edges and adding them one by one if they don't form a cycle.

How it Works:

  1. Sort all edges by weight in ascending order
  2. Initialize a disjoint-set (Union-Find) for all vertices
  3. For each edge in sorted order:
    • If the edge connects two different sets:
      • Add the edge to MST
      • Union the two sets
    • Otherwise, skip it (would create a cycle)
Time Complexity: O(E log E) or O(E log V)
Space Complexity: O(V)

Applications:

  • Network design optimization
  • Circuit design
  • Transportation networks

Key Data Structure - Union-Find:

Union-Find (Disjoint Set Union) efficiently tracks which vertices are in the same connected component:

  • Find: Determine which set a vertex belongs to (with path compression)
  • Union: Merge two sets (with union by rank)
